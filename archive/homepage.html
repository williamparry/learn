What if I want all my content to share the same font?

You could simply add the font-family property to every definition, but really this is where we use the "Cascading" part of CSS. To do this you'll need to find the top-most tag in your document you want the style to apply to. Let's go up the document tree:

  |
  |

     |
     + header
     |   |
     |   |
     |   - h1
     |
     |
     - #introduction
     |
     |
     - h2
     |
     |
     - p

While you're in Chrome Developer tools you can check out a CSS concept called "display". CSS has couple of main types of element block-level and inline .

tl;dr: if it's block-level the element moves onto its own line and fills the width of the parent element (like an [html]

[/html] tag) and if it's inline it will fill only its own space (like an [html][/html] tag).

You can see block-level elements in practice by inspecting a title (say "About me") and noticing how although the text is only short, the bounding box goes all the way to the end. Similarly, to view an "inline" element you can inspect the contact link and notice how the bounding box is in line with the rest of the text.

You can override these in CSS – try giving the link a display of "block" and see how it goes onto another line (extra points if you can do this in developer tools).

Moving on, and we've got a concern with our images now: what if they don't load (bad connection, typo etc)? We can get provide an alt attribute that will show up in its place to describe the image.

It's best practice to provide an alternate text for images that are important for your content. If it's not important the image should be a background or have an empty alt attribute.
