Writing CSS

A CSS file is a compilation of definitions. Think of them as instruction sets of how an HTML element (such as a paragraph) should look. Here's an example of one definition that targets the

tag.

p {
	color: red;
}

Looks very different to HTML, doesn't it? This is because they serve two purposes: HTML is there to mark up a semantic document, and CSS is there purely to make the document look more aesthetically pleasing.

If you put that CSS into your style.css file, save it and then refresh the browser you won't believe what it will look like.

Want a sneak peek?

Ok, maybe you guessed that the font would turn red. Is it that easy? In some ways it is that easy. Try changing the font to blue and once you're done playing with colours (oh, and use american style spelling, so "colors") let's go over the structure of a CSS definition.

First we have the selector, which is our target for whatever style we want to add. In this case it's a

tag.

p

Then we follow it with a curly brace, indicating that we're opening up a new definition:

p {

To close the definition we add a closing curly brace. It's best to close the definition now and fill it in afterwards do it now because we might forget later; CSS is similar to HTML in the way that it will quietly mess up the page without telling you.

p {

}

We now have an empty CSS definition that's targeting all

tags in our document. Let's override the standard font size the browser gave us (16px) to make it huge (just because we can).

p {
	font-size: 48px;
}

When you refresh your browser you're going to see huge paragraph text:

For the purposes of this course we'll be using the px (pixel) measurement for fonts (see more on this ).

You'll notice a similar structure between making the color red and the font size large. Once we're inside the curly braces we can start loading up the selector (in this case the p) with as many styles as we need.

First we define the property we want to add: 

p {
	font-size
}

Add a colon, so that it can distinguish between the property and value:

p {
	font-size:
}

Now we can give it a value:

p {
	font-size: 48px
}

And close it off with a semi-colon so that it knows we've finished with that property:

p {
	font-size: 48px;
}

That is a fully formed CSS definition. But what if we want to make the font huge and red? Have a go!

Stuck? Here's what your CSS definition should look like:

p {
	font-size: 48px;
	color: red;
}

You can imagine what that's going to look like in your browser.
